# Требования к функционалу фильтрации статей

## Общие требования
Необходимо предусмотреть возможность фильтрации статей по двум критериям:
1. **Sources** (Источники)
2. **Topics** (Темы/Разделы)

Фильтрация должна быть реализована в виде отдельных экранов для каждого критерия.

## Пользовательский интерфейс (UI)

### Экраны фильтрации
*   **Экран Sources**: Отдельный экран для выбора источников.
*   **Экран Topics**: Отдельный экран для выбора тем.

### Компонент выбора (Таблица)
Для выбора значений на обоих экранах должна использоваться `rich` таблица с следующими характеристиками:
*   **Paging**: Поддержка постраничной навигации, аналогично основной таблице со статьями.
*   **Multi-selection**: Возможность выбора нескольких значений одновременно.
*   **Выделение**: Выбранные строки должны выделяться цветом фона (background) для наглядности.
*   **Управление выбором**: Выбор и снятие выбора для текущей строки осуществляется нажатием клавиши `Enter`.

### Индикация фильтров
*   Если применен фильтр по Sources и/или Topics (выбрано хотя бы одно значение), этот факт должен быть **отражен в заголовке** приложения Info Radar.
*   Состояние фильтрации должно быть **однозначно понятно** пользователю при взгляде на основной экран.

## Управление (Shortcuts)

*   `s` — Открыть экран фильтра по Sources.
*   `t` — Открыть экран фильтра по Topics.
*   `Esc` — Сбросить фильтр для текущего экрана (отдельно для Sources, если открыт экран Sources, и отдельно для Topics, если открыт экран Topics).

## Логика взаимодействия

*   **Зависимость Topics от Sources**:
    *   Если пользователь выбрал и применил фильтр по Sources, то список доступных тем на экране фильтрации по Topics должен быть отфильтрован.
    *   Пользователь должен видеть только те Topics, которые относятся к выбранным Sources.
