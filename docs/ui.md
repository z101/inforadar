# UI

## Обзор

Считается, что UI построен на концепции независимых экранов, связанных логикой переходов. Экран занимает всю ширину и высоту консоли.

Есть 3 типа экранов:
1. View Screen - это основные экраны с таблицами для отображения информации из БД. В header отображается заголовок экрана/ У header стиль шрифта жирный, а у основного текста таблицы - обычный. Данные выводятся блоками: в одном блоке столько строк, сколько помещается на страницы. ВАЖНО! Так как в ячейках отображается весь контекст без переноса слов, то высота строк может быть разной. Необходимо явно просчитывать общую высоту строк содержимого таблицы так, чтобы строки помещались точно на 1 экран (с учетом title приложения, header и footer таблицы). В footer пишется диапазон выводимых строк и общее количество строк (Items). Выводимые на странице строки нумеруются с 1 в столбце #. Базовые команды: h - возврат к предыдущему экрану, j/k - переход к следующему/предыдущему блоку, g/G - переход к первой/последней странице, l - переход к следующему экрану, номер строки + Enter - переход к Action Screen с превью статьи (причем при вводе номера строки выделенная строка должна изменять цвет текста на зеленый, приложение должно позволять выбирать только номера тех строк, которые находятся в текущем блоке и игнорировать числа, не входящие в этот диапазон), q - выход из приложения
2. Action Screen - это связанные с View Screen экраны (1 View - multiple Action), которые содержат действия с данными View Screen. Esc - возврат к предыдущему View Screen
3. Help Screen - это экраны для отображения справки (краткое описание экрана, полный перечень доступных команд), который вызывается по шорткату "?". Связаны с View и Action Screen 1-к-1. Esc - возврат к предыдущему экрану

## Структура экранов

### Стартовый View Screen

При старте приложения должен открываться стартовый View Screen со списком статей всех источников, отсортированных по умолчанию по дате публикации.

Заголовок: Info Radar \[Articles\].

Таблица содержит столбцы: Source, Topic, #, Article, Date, Details. Количество столбцов должно зависеть от ширины консоли. На столбец Article должно приходиться наибольшее количество символов, но не меньше 40. Минимальные отображаемые столбцы: Source, #, Article, Date. Если ширина позволяет, то отображаются все столбцы.

Команды:
- r - Action Screen с логикой чтения статей из источников из интернета
- f - Action Screen с логикой фильтрации статей на соответствующем View Screen (факт фильтрации должен быть отмечен маркером в footer)
- s - Action Screen с логикой сортировки статей на соответствующем View Screen
