Мы с тобой Python Developers. Нам нужно спроектировать (консольное) приложение "Информационный радар" (inforadar), которое будет помогать получать последние интересные (технические) статьи для чтения пользователем из разных источников информации (новостные сайты, сайты со статьями, посты из Telegram и, возможно, прочее).

Цель данного предложения - получить все недавние ссылки на статьи/посты и их метаданные (например, автор, количество просмотров, и пр.) из этих заранее определенных источников (считается, что пользователь будет запускать приложение на регулярной основе, нужно уметь выделять новые статьи и сохранять уже собранные), и предоставить пользователю удобный интерфейс, чтобы он мог быстро выгрузить непрочитанные (новые статьи), мог добавить, изменить (сконфигурировать) и удалить источники, у пользователя должна быть возможность отмечать прочитанные и интересные статьи, фильтровать и выгружать данные по ним, иметь возможность быстро переходить по ссылкам нужных статей.

Предложи архитектуру приложения, сценарии взаимодействия с пользователем (особое внимание удели удобству использования), продумай формат хранения данных (openxml, sqlite или еще что-то). Мы пока проектируем, пока не пиши код. На первом шаге нам нужно упорядочить список требований и продумать верхнеуровневую архитектуру. Напиши свои варианты и обоснуй, почему по твоему мнению они больше всего подходят к нашей задаче.

===

Выбираем вариант Python scripts (простой запуск на локальной машине) - для логики и SQLite для хранения конфигурации и данных по источникам и статьям. WebUI пока не рассматриваем, управление через консоль (CLI). Приложение должно запускаться через командную строку (терминал) как:

ir <command> <parameters>

где ir - сокращение от info radar.

Команды первой версии:
 - fetch. Выбор новых статей по источникам. По умолчанию, без параметров (опрос по всем источникам). Параметр - имя источника (или источников через запятую) - обновление только по указанным источникам
 - export. Выгрузка в openxml Excel file данных по статьям для ознакомления и правки пользователем. Параметры:
    > --output (-o). Имя выходного файла. По умолчанию - "data/articles.xlsx"
    > --threshold (-t). Маркер отсечки (last_marker в твоем тексте выше). По умолчанию не задан.
    > --unread (-u). Switch параметр, который выбирает только статьи, которые не помечены, как read
    > --interesting (-i). Switch параметр, который выбирает только статьи, которые помечены, как interesting
 - import. Загрузка файла с параметрами read и interesting обратно в базу. Параметры:
    > --output (-o). Имя выходного файла. По умолчанию - "data/articles.xlsx"
 - stat. Вывод количества статей в базе данных (по источникам в виде таблицы: имя источника - общее количество статей - количество статей, помеченных как прочтенные (общее количество + процент от общего числа) - количество статей, помеченных как прочтенные (общее количество + процент от общего числа))

Нужно отдельно продумать, как хранить в БД конфигурацию конкретных источников, учитывая, что набор параметров конфигурации может быть разным. Например, такой вариант:
Source | Param | Value

Также надо отдельно продумать, как хранить статьи по разным источникам, учитывая, что у таких статей могут быть как общие, так и различающиеся столбцы.

Команда export должна выводить единый файл со всеми возможными столбцами сохраненных статей (по фильтрам), причем одноименные столбцы схлопываются, а разноименные выводятся отдельно, только если хотя бы в одной строке есть данные. Иначе пропускаются.

Согласно постановке выше предложи организацию python кода по файлам и папкам и структуру таблиц базы данных SQLite.
