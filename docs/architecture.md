# Архитектура "Информационного радара"

## Обзор
Модульное консольное приложение с разделением ответственности на слои: интерфейс, бизнес-логика, работа с данными и интеграция с внешними источниками.

## Потоки данных

### 1. Сбор данных (Fetch)
`CLI -> Core Engine -> Provider (RSS/Scraping) -> Storage`
Core Engine запрашивает у провайдера новые статьи. Провайдер получает их через RSS и обогащает скрапингом. Результат сохраняется в БД.

### 2. Актуализация (Refresh)
`CLI -> Core Engine -> Storage (Get) -> Provider (Enrich) -> Storage (Update)`
Core Engine получает список устаревших статей из БД, запрашивает свежие метаданные у провайдера и обновляет записи в БД.

### 3. Просмотр (View/Triage)
`CLI -> Core Engine -> Storage`
CLI запрашивает отфильтрованный список статей из БД через Core Engine для отображения пользователю.

## Компоненты

### 1. CLI (`main.py`)
*   **Роль:** Точка входа и пользовательский интерфейс.
*   **Стек:** `Typer` (команды), `Rich` (таблицы, интерактив).
*   **Функции:** Обработка команд пользователя, отображение данных, интерактивный режим сортировки.

### 2. Core Engine (`core.py`)
*   **Роль:** Оркестратор бизнес-логики.
*   **Функции:**
    *   Управление процессом сбора (`run_fetch`) и обновления (`run_refresh`).
    *   Связывание провайдеров с хранилищем.
    *   Предоставление API для CLI.

### 3. Providers (`providers/`)
*   **Роль:** Адаптеры к внешним источникам.
*   **Реализация:** `HabrProvider` (`habr.py`).
*   **Функции:**
    *   Парсинг RSS.
    *   Скрапинг HTML-страниц (получение текста, рейтинга, комментариев).
    *   Нормализация данных в формат `Article`.

### 4. Storage (`storage.py`)
*   **Роль:** Абстракция над базой данных.
*   **Стек:** `SQLAlchemy`, `SQLite`.
*   **Функции:** CRUD-операции для статей, управление транзакциями, фильтрация выборок.

### 5. Models (`models.py`)
*   **Роль:** Описание схемы данных.
*   **Сущности:** `Article` (ORM-модель таблицы статей).

### 6. Config (`config.yml`)
*   **Роль:** Внешняя конфигурация.
*   **Настройки:** Список хабов, шаблоны URL, параметры глубины сбора.
